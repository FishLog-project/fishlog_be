# geo.md — 낚시 스팟 위치·주변 검색

> 낚시 스팟의 좌표 저장과 위치 기반(반경/지도) 검색 설계. 현재 **📋 전부 계획 단계**.

## 요구사항
- 낚시 스팟을 위경도 좌표로 저장.
- 사용자 위치/지도 영역 기준 **주변 스팟 검색**(반경 N km, 또는 지도 bounding box).
- 지도 표시·지오코딩은 외부 지도 API 연동 → `docs/external.md`.

## 저장 방식 (선택지, 확정 필요)
- **MySQL 공간 데이터(권장)**: `POINT` 타입 + `SPATIAL INDEX`, `ST_Distance_Sphere`/`MBRContains`로 반경·영역 질의.
- **단순 위경도 컬럼 + 계산식**: `lat`/`lng` DOUBLE 컬럼 + Haversine 계산(인덱스 효율 낮음, 소규모엔 무난).

## 질의 패턴 (초안)
- 반경 검색: 중심 좌표 + 거리 → 스팟 목록(가까운 순 정렬).
- 영역 검색: 지도 4개 좌표(bbox) → 화면 내 스팟.
- 성능: 스팟 수 규모에 따라 공간 인덱스 필요 여부 결정.

## 확정 필요 항목
- [ ] MySQL 공간 타입 사용 여부 (JPA/Hibernate spatial 의존성 추가 여부)
- [ ] 좌표계(SRID 4326) 및 거리 계산 방식
- [ ] 스팟 데이터 규모(인덱스 전략에 영향)
