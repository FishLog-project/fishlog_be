# product.md — 도메인·비즈니스 규칙

> 서비스 설명 기반 **도메인 초안**입니다. 세부 규칙(승인 흐름, 레벨/뱃지 체계 등)은 대부분 **📋 TBD**이며 확정되는 대로 채웁니다.

## 서비스 개요

fishlog는 **관광 + 낚시 레저 통합 플랫폼**입니다.

- 전국 **낚시 스팟**을 제공하고, 각 스팟에서 잡히는 **어종** 정보와 **주변 관광 시설** 정보를 함께 노출합니다.
- 사용자는 낚시 스팟을 중심으로 **낚시 여행**을 계획·소비합니다.
- **어종 도감**: 잡은 물고기를 **사진으로 인증**하며 자신만의 도감을 채워 나가는 **게임적(수집) 요소**를 제공합니다.

## 핵심 도메인

### 1. 낚시 스팟 (Spot)
- 전국 낚시 포인트. 위치(좌표), 유형(갯바위/방파제/선상 등), 대상 어종 등을 보유. 📋
- **데이터 출처:** 공공데이터포털 바다낚시지수 API(15142486)에서 수집한 **49개 고유 위치**(갯바위·선상). 이름·위도·경도만 시드 저장 → `docs/external.md` §1, `docs/geo.md`, `docs/spec.md`. 🚧 (추후 추가 가능)
- 위치 기반 **주변 검색**(반경/지도) 제공 → `docs/geo.md`. 📋

### 2. 어종 (Fish)
- 어종 도감의 기준 데이터. 이름·이미지·서식/제철·특징 등. 📋
- 스팟과 다대다 관계(스팟에서 잡히는 어종). 📋
- **어종 카탈로그 규모 📋 TBD:** 바다낚시지수 API는 대상어종을 **6종**(감성돔·농어·돌돔·벵에돔·우럭·참돔)만 제공 → 도감 콘텐츠로는 부족. catch-all `기타어종`이 포괄하는 실제 어종을 **별도 조사**한 뒤 도감(`fishes`)을 확장할 계획(어종 마스터는 이 API와 분리해 확장하는 방향). `기타어종`의 도감 대상 여부도 함께 확정 → `docs/spec.md`, `docs/external.md` §1.

### 3. 어종 도감 / 인증 (Collection)
- 사용자가 잡은 물고기를 **사진으로 인증**하면 개인 도감에 어종이 등록/해금됨. 📋
- **인증 승인 방식(자동 vs 관리자 수동), 중복 인증 처리, 인증 취소/신고** 규칙: 📋 TBD (사용자 확인 필요).
- **게이미피케이션**(레벨/경험치/뱃지/도감 완성도 등): 📋 TBD.
- 사진 저장·처리 → `docs/media.md`.

### 4. 관광 시설 (Tour)
- 낚시 스팟 주변 관광/편의 시설 정보. 외부 관광 API 연동 → `docs/external.md`. 📋

### 5. 회원 (User)
- 자체 이메일/비밀번호 가입 + JWT 인증 → `docs/security.md`. 📋
- 개인 도감·인증 이력·여행 계획의 소유 주체.

## 사용자 시나리오 (초안)
1. 스팟 탐색 → 지도/주변 검색으로 낚시 스팟 찾기
2. 스팟 상세 → 대상 어종·주변 관광 시설·날씨/물때 확인
3. 낚시 후 → 잡은 물고기 사진 업로드로 어종 인증
4. 도감 → 인증한 어종이 도감에 채워지고 수집 진행도 상승
