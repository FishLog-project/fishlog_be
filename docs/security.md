# security.md — 인증·인가

> 방향은 **자체 이메일/비밀번호 로그인 + JWT**로 확정. 세부 구현은 **📋 TBD**.

## 인증 방식
- **자체 회원가입/로그인**: 이메일 + 비밀번호. 비밀번호는 해시 저장(BCrypt 권장). 📋
- **JWT 발급**: 로그인 성공 시 Access(+ Refresh) 토큰 발급. 📋
  - 토큰 만료·갱신 정책, 서명 시크릿 관리(환경변수 → `be_config`) 확정 필요.

## 인가 (엔드포인트 정책)
- **공개(인증 불필요):** `GET /api/health`, 회원가입/로그인, 낚시 스팟·어종 조회 등 열람성 API (초안). 📋
- **보호(인증 필요):** 어종 도감 인증, 개인 도감·여행 계획 등 사용자 소유 리소스. 📋
- **관리자:** 어종/스팟 데이터 관리, (필요 시) 인증 사진 승인. 📋

## 구현 메모 (계획)
- Spring Security + JWT 필터 체인, `global/security` 패키지에 배치 (`docs/architecture.md`).
- 인증 주체(`User`)는 `docs/product.md`의 회원 도메인과 연결.

## 확정 필요 항목
- [ ] 토큰 구조(Access/Refresh)·만료·저장 위치(클라이언트)
- [ ] 비밀번호 정책·재설정 흐름
- [ ] 이메일 인증 여부(가입 시 확인 메일 등)
- [ ] 권한(Role) 체계 (USER/ADMIN 등)
